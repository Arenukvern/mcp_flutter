# Feature Map

> Tool names below show the bare catalog/CLI form. MCP `tools/call` adds the
> `core_` capability prefix (v3.0.0+) â€” e.g. `core_tap_widget`,
> `core_hot_reload_and_capture`. CLI `flutter_mcp_cli exec --name <bare>`
> uses the unprefixed form.

## Core Features

| Feature | Why It Is Useful | Learn More |
| --- | --- | --- |
| App error retrieval (`get_app_errors`) | Gives condensed runtime errors for faster fixes | [Built-in tools](/core/built_in_tools) |
| Screenshots (`get_screenshots`) | Lets AI see current UI state | [Built-in tools](/core/built_in_tools) |
| View details (`get_view_details`) | Exposes view metadata and widget-tree inspection payloads | [Built-in tools](/core/built_in_tools) |
| Target discovery (`discover_debug_apps`) | Returns canonical websocket target URIs for reliable multi-app selection | [Built-in tools](/core/built_in_tools) |
| Unified evidence (`capture_ui_snapshot`) | Captures screenshot/layout/error data in a single debugging bundle | [Built-in tools](/core/built_in_tools) |
| Coordinate inspection (`inspect_widget_at_point`) | Maps visual coordinates back to widget/render nodes and source hints | [Built-in tools](/core/built_in_tools) |
| Interaction layer (`semantic_snapshot` + `tap_widget` / `enter_text` / `scroll` / `swipe` / `long_press` / `drag`) | Lets an agent drive the running app like a user (Playwright-style refs, two-tier semantic-action â†’ pointer-event dispatch) | [Built-in tools](/core/built_in_tools), [Interaction cookbook](/guides/interaction_cookbook) |
| Fused edit/preview (`hot_reload_and_capture`) | Single round-trip: hot reload, screenshot, fresh semantic snapshot, and errors | [Built-in tools](/core/built_in_tools) |
| Runtime introspection (`evaluate_dart_expression`, `get_recent_logs`) | Read app state and recent logs without pre-registered tools | [Built-in tools](/core/built_in_tools) |
| Hot reload / hot restart | Fast iteration from assistant commands | [Built-in tools](/core/built_in_tools) |
| Dynamic tool registry | Register project-specific tools at runtime | [Dynamic tool registry](/core/dynamic_tools_registry) |
| Connection targeting | Reliable tool execution with multiple debug targets | [MCP configuration](/core/mcp_configuration) |
| CLI + MCP dual interface | Use terminal automation and assistant workflows with shared core behavior | [CLI vs MCP](/start_here/cli_vs_mcp) |
| CLI command recipes | Copy-paste patterns for local and CI automation | [CLI quick recipes](/start_here/cli_quick_recipes) |
