# Human Path: Client Setup

Use this page when you want to configure AI clients yourself.

## Shared Rules For All Clients

- Keep other existing MCP servers in your config.
- Add or merge only the `flutter-inspector` entry.
- Use an absolute path to `mcp_server_dart/build/flutter_inspector_mcp`.

## Codex

```json
{
  "mcpServers": {
    "flutter-inspector": {
      "command": "/ABSOLUTE/PATH/mcp_flutter/mcp_server_dart/build/flutter_inspector_mcp",
      "args": [
        "--dart-vm-host=localhost",
        "--dart-vm-port=8181",
        "--resources",
        "--images",
        "--dynamics"
      ],
      "env": {},
      "disabled": false
    }
  }
}
```

## Cursor

Use `--no-resources` because Cursor resource handling may be limited.

```json
{
  "mcpServers": {
    "flutter-inspector": {
      "command": "/ABSOLUTE/PATH/mcp_flutter/mcp_server_dart/build/flutter_inspector_mcp",
      "args": [
        "--dart-vm-host=localhost",
        "--dart-vm-port=8181",
        "--no-resources",
        "--images",
        "--dynamics"
      ],
      "env": {},
      "disabled": false
    }
  }
}
```

## Cline / RooCode

- Prefer `--no-resources`.
- For image files instead of inline image responses, add `--save-images`.

## Validation Checklist

After restart of your client:

1. Ask for available tools.
2. Ask for screenshot.
3. Ask for app errors.

If you get `connection_selection_required`, retry with `arguments.connection.targetId` from the returned `availableTargets`.
